          本章概览

          本章主要讲解抽象类和接口。在上一章主要介绍了在继承中的多态，提到过多态塑造了一个整齐的对于继承类体系的API。而在这一章，主要探讨为了使 jar 包 API 的使用更加方便、可控、易于组合和拓展，Java所创造的一些关键字和机制：抽象类和抽象方法、接口。抽象类是一个单纯用作API的基类，这个类提供了需要实现的抽象方法，而其导出类则提供了这些抽象方法的实现。

          更进一步，Java提供了一个叫做接口的东西，接口完全脱离了类，其提供了一套方法标准，任何类都可以生成其实现了接口，其需要提供接口所定义方法的实现。接口最重要的作用是在使用“组合和继承的结合”中，在结合类（下称客户端）中解决由于实例化类使用而造成的客户端和调用的导出类方法过于耦合的问题。我们首先需要定义一套在客户端使用的接口，然后重构包，使用接口+抽象类来提供一个清晰的、符合定义的接口，也可以使用适配器来粘合接口和包中基类的方法。

          接口的功能强大，可以通过接口继承来构建接口体系。可以通过适配器（用作旧类的扩展）、直接构造（用作新类的扩展）来适配接口来让我们的类通过系统提供的接口来实现意想不到的调用，以扩展类的能力。在本章的最后，提到了使用接口来进一步彻底解决客户端耦合的问题——工厂模式。工厂模式提供客户端完全抽象的一套逻辑，客户端代码不用更改，只用修改工厂生成的对象即可实现代码拓展。
