      本章概览

      本章首先介绍了标记文件，这是一种用标记来进行页面包含的标准。其次介绍了自定义标记和标记处理器，有了自定标记，我们就可以进行更为方便的、更加解耦的开发。虽然EL+JSTL能提供大多数的View层的逻辑和计算操作，但是当我们希望新建一个Java对象，在数个Java对象之间执行一系列动作，最后获取输出的时候，自定标记就可以派上用场了。区别于EL的静态函数，自定标记可以传递属性和体，并且可以动态调用变量格式化输出，其功能更为强大。

      在这篇文章中的第一部分，介绍了使用标记文件简化包含以及参数传递。第二部分主要介绍了简单标记处理器以及自定标记，以及父子标记的互相调用和动态属性实现。自定标记可以弥补EL+JSTL对于显示页面的逻辑上的不足，但是，也不能滥用，对于模型应该做的事情，应该交给模型来实现逻辑。此外，近年的前后端分离也导致了标准动作+EL+JSTL+自定标签这种解决方案用的越来越少。
